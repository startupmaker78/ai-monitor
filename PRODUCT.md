# Вебмонитор — Product Definition

> Источник правды по продукту. Обновляется при каждом значимом изменении концепции.

---

## Суть продукта

Вебмонитор — SaaS-сервис для владельцев сайтов. Трекинг-скрипт записывает поведение посетителей, Яндекс.Метрика подтягивает ключевые метрики, а Claude AI анализирует данные и выдаёт конкретные рекомендации для роста конверсии, снижения отказов и увеличения времени на сайте.

**Домен**: вебмонитор.рф

---

## Стратегия по платформам

**Первая итерация — фокус на Тильде.**
Начальная аудитория привлекается через базу дизайнеров и верстальщиков nolim.cc (~4 500 человек). Они работают с Тильдой, поэтому онбординг и инструкция по установке скрипта написаны для Тильды.

**Архитектура — универсальная с первого дня.**
Трекинг-скрипт работает на любом сайте: Тильда, WordPress, Битрикс, кастомные сайты, React-приложения. Скрипт — одна строка кода, которую можно вставить в любой HTML. Технических ограничений нет.

**Следующие итерации** — инструкции по установке и кейсы для других платформ.

## Типы пользователей

### Подрядчик (дизайнер, верстальщик, маркетолог)
- Регистрируется самостоятельно
- Добавляет клиентов (владельцев сайтов) в свой кабинет — только через приглашение
- Получает **50% комиссию** с каждой оплаты своего клиента
- Видит список клиентов, их статусы, свой заработок и может вывести средства
- При регистрации видит **демо-стенд** с примером заполненного дашборда

### Владелец сайта
- Получает приглашение на email от подрядчика
- Заходит по временному паролю, при первом входе меняет его
- Видит метрики своего сайта, рекомендации AI и аналитику динамики
- **Самостоятельная регистрация не предусмотрена** — только через приглашение

---

## Процесс приглашения клиента

1. Подрядчик заполняет форму: URL сайта, название компании, email владельца, телефон владельца, ID Яндекс.Метрики
2. Система создаёт аккаунт владельца и отправляет ему email со ссылкой и временным паролем
3. Владелец переходит по ссылке, меняет пароль, оказывается в своём дашборде
4. Подрядчик видит клиента в своём списке со статусом «Активен»

## Ключевые фичи MVP

### 1. Трекинг-скрипт
- Одна строка кода — вставляется в `<head>` любого сайта
- Базируется на **rrweb** (запись сессий: клики, скролл, движения мыши)
- Хранение данных сессий — **30 дней**, затем автоудаление
- Маскировка ввода в поля `password`, `email`, `tel` (обязательно, до отправки)
- IP-адреса хранятся только в **хэшированном виде** (требование 152-ФЗ)
- В MVP: инструкция по установке написана для Тильды

### 2. Интеграция с Яндекс.Метрикой
- Подключение через API-ключ (вводится вручную в настройках)
- Подтягиваем: визиты, уникальные посетители, цели, конверсия, отказы, время на сайте
- Ежедневная синхронизация, кнопка «Обновить сейчас»

### 3. AI-анализ (Claude API)
- Запускается вручную — **владельцем сайта или подрядчиком** (когда подрядчик открыл дашборд клиента)
- Лимиты считаются по тарифу клиента (владельца), независимо от того кто нажал кнопку
- Анализирует записанные сессии + метрики Яндекс.Метрики
- Возвращает **10–20 рекомендаций**, отсортированных по приоритету:
  - 🔴 Критично
  - 🟡 Важно
  - 🟢 Хорошо (уже работает)
- В интерфейсе по умолчанию показываются **топ-10** — остальные скрыты под кнопкой «Показать ещё N рекомендаций»
- В базе данных хранятся **все** рекомендации
- Месячный лимит по тарифу. Счётчик сбрасывается в начале каждого платёжного периода. 24-часового кулдауна нет.
- История всех анализов сохраняется

### 4. Статусы рекомендаций
Каждая рекомендация проходит жизненный цикл:
- **NEW** — только что появилась после анализа
- **IN_PROGRESS** — владелец нажал «Принять», взял в работу
- **DONE** — владелец нажал «Выполнено» (правка реально внесена на сайте)
- **REJECTED** — владелец нажал «Отказаться»; опционально вводит причину отказа

Все статусы видны в интерфейсе — прозрачная картина что принято, что в работе, что сделано, от чего отказались и почему.

### 5. Дашборд владельца
- Текущие метрики сайта (конверсия, отказы, время на сайте, визиты)
- Список рекомендаций с фильтрацией по приоритету и статусу
- **Сравнение по произвольным периодам** (подробнее ниже)
- Эффект от каждой выполненной рекомендации

### 6. Аналитика динамики — сравнение по периодам
В дашборде: выбор произвольного периода для сравнения метрик «до и после».

Доступные варианты:
- «С последних правок» (с даты последней рекомендации со статусом DONE)
- «С выбранной даты» (date picker)
- «За последний месяц / 3 месяца / год»
- «Между двумя произвольными датами»

Система ежедневно сохраняет снапшот всех метрик сайта (MetricsSnapshot) и может сравнивать значения на любые две даты.

### 7. Аналитика по каждой рекомендации
Для каждой рекомендации со статусом DONE система показывает её собственный эффект:
- Мини-блок «до/после» прямо на карточке рекомендации
- «До» = снапшот метрик на момент принятия рекомендации (IN_PROGRESS)
- «После» = снапшот метрик через 7 дней после выполнения (DONE)
- Общая таблица «Эффект по рекомендациям» — сортировка по наибольшему росту

### 8. Кабинет подрядчика
- **Демо-стенд**: после регистрации в списке клиентов автоматически появляется демо-проект «Демо-стенд Вебмонитор» — заполненный дашборд со всеми фичами. Кнопка «Скопировать ссылку на демо» → подрядчик отправляет ссылку потенциальному клиенту, тот видит продукт без регистрации.
- Список реальных клиентов и их статусы
- Сводка заработка (начислено, выплачено, доступно к выводу)
- Форма добавления нового клиента (приглашение)
- Запрос на вывод средств

### 9. Дашборд клиента в кабинете подрядчика
Когда подрядчик кликает на клиента в своём списке — открывается дашборд этого клиента с двумя вкладками:

**Вкладка «Вид владельца»**
Подрядчик видит ровно то, что видит сам владелец: метрики, рекомендации, аналитика периодов, история событий. Позволяет объяснять клиенту что происходит и запускать анализ от его имени.

**Вкладка «Инструменты подрядчика»** *(только для подрядчика, владелец её не видит)*
- Статус доступа клиента (заходил ли, когда последний раз)
- Готовый код скрипта для установки (с токеном клиента, один клик — скопировать)
- Статус интеграции с Яндекс.Метрикой (подключена / не подключена, ошибки)
- Быстрые действия: «Написать клиенту», «Отправить PDF-отчёт», «Открыть сайт клиента»

### 10. Контакт с подрядчиком (в дашборде владельца)
В шапке дашборда владельца постоянно отображается блок с подрядчиком:
- Аватар и имя подрядчика
- Кнопка «Написать» → открывает модальное окно с вариантами связи: WhatsApp, Telegram, Email
- Контактные данные подрядчик вводит при регистрации (телефон, Telegram-ник, email)

Цель: владелец всегда знает к кому обратиться с вопросом, подрядчик остаётся «видимым» партнёром.

### 11. История событий (в дашборде владельца)
Хронологическая лента всех значимых событий по сайту:

- Подключён счётчик Яндекс.Метрики
- Установлен трекинг-скрипт (первая сессия)
- Запущен AI-анализ (дата, количество рекомендаций)
- Рекомендация взята в работу (название)
- Рекомендация выполнена (название)
- Рекомендация отклонена (название, причина)
- Изменены настройки сайта
- Оплата тарифа (план, период)

Каждая запись: дата + короткое описание + иконка типа события. Это не дублирует аналитику по рекомендациям — это именно лог событий для понимания «что вообще происходило с сайтом».

## Юридические требования

- **152-ФЗ**: все данные пользователей хранятся на серверах в РФ (Yandex Cloud, регион ru-central1)
- **Маскировка PII**: поля `password`, `email`, `tel` маскируются на стороне клиентского скрипта до отправки на сервер
- **IP-адреса**: хэшируются (SHA-256 + соль) до записи в базу, оригинал не сохраняется
- **Юрлицо**: ООО «Супер Просто» — используется для подключения ЮКассы и всех коммерческих операций
- Необходимо пользовательское соглашение и политика конфиденциальности
